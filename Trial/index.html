how to select elements based on node type

You can select elements based on their node type using the TreeWalker API in JavaScript. This is especially useful for selecting non-element nodes like text nodes or comments, as querySelector and querySelectorAll only work with elements.

Here's how you can do it:

JavaScript 
function getNodesByType(rootNode, nodeType) {
  const nodes = [];
  const treeWalker = document.createTreeWalker(
    rootNode,
    nodeType, // The node type to filter for
    null,     // No custom filter function
    false     // No entity expansion
  );

  let currentNode;
  while (currentNode = treeWalker.nextNode()) {
    nodes.push(currentNode);
  }

  return nodes;
}

// Example usage: Get all text nodes within the document body
const textNodes = getNodesByType(document.body, NodeFilter.SHOW_TEXT);
textNodes.forEach(node => {
  console.log("Text Node:", node.nodeValue.trim());
});

// Example usage: Get all comment nodes within the entire document
const commentNodes = getNodesByType(document.documentElement, NodeFilter.SHOW_COMMENT);
commentNodes.forEach(node => {
  console.log("Comment Node:", node.nodeValue.trim());
});

// Example usage: Get all element nodes within the document body (equivalent to document.body.querySelectorAll('*'))
const elementNodes = getNodesByType(document.body, NodeFilter.SHOW_ELEMENT);
elementNodes.forEach(node => {
  console.log("Element Node:", node.tagName);
});

//Example usage: Get all processing instruction nodes
const processingInstructionNodes = getNodesByType(document, NodeFilter.SHOW_PROCESSING_INSTRUCTION);
processingInstructionNodes.forEach(node => {
  console.log("Processing instruction Node:", node.nodeName, node.nodeValue);
});

Explanation:

  1. document.createTreeWalker(rootNode, nodeType, null, false): This creates a TreeWalker object.

    • rootNode: The starting node for the traversal (e.g., document.body, document.documentElement, or any other node).
    • nodeType: This is the crucial part. It specifies the type of nodes to select. You use constants from the NodeFilter object:
      • NodeFilter.SHOW_ALL: Selects all node types.
      • NodeFilter.SHOW_ELEMENT: Selects element nodes (like <div>, <p>, <span>, etc.).
      • NodeFilter.SHOW_TEXT: Selects text nodes (the actual text content within elements).
      • NodeFilter.SHOW_COMMENT: Selects comment nodes (``).
      • NodeFilter.SHOW_DOCUMENT: Selects the document node itself.
      • NodeFilter.SHOW_DOCUMENT_TYPE: Selects the document type node (<!DOCTYPE html>).
      • NodeFilter.SHOW_DOCUMENT_FRAGMENT: Selects document fragment nodes.
      • NodeFilter.SHOW_PROCESSING_INSTRUCTION: Selects processing instruction nodes (e.g. <?xml-stylesheet ... ?>).
    • null: A filter function (not needed in this basic example).
    • false: Whether to expand entity references.

  2. treeWalker.nextNode(): Moves the TreeWalker to the next node of the specified type.

  3. Accessing Node Properties:

    • node.nodeValue: For text nodes and comment nodes, this gives you the text content.
    • node.tagName: For element nodes, this gives you the tag name (e.g., "DIV", "P", "SPAN").
    • node.nodeName and node.nodeValue: For processing instruction nodes, those properties provide the name (target) and the content of the instruction.
